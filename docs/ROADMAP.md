## Year 1 — Foundations

**Goal**: Build the SCC corpus, construct knowledge graphs, and extend NXS+ with multi-vault loading and embedding-based resonance detection.

### M1–M3: Corpus and Ontology
- Corpus selection from SCC sources: research papers, course documents, student projects, faculty publications, exhibition texts, artist statements, design process notes, archival resources
- Metadata schema design: domain, medium, method, historical period, conceptual themes, epistemic function
- Seed ontology design: class hierarchy for SCC knowledge domains
- Graph granularity decisions: how fine-grained should nodes be for effective collision?

### M4–M6: Graph Construction
- Concept graph extraction and relation typing
- Method graph: techniques, procedures, analytical and making practices
- Actor graph: researchers, artists, labs, institutions, collaborations
- Provenance model: every node traceable to its source document
- Human-supervised throughout (Kommineni et al., 2024 pipeline adapted for cultural material)

### M7–M9: NXS+ Extension
- **Multi-vault loader**: modify `loadVault()` to accept a second JSON, assign discipline colour-coding, render both entity arrays in shared scene with origin offset
- **Embedding-based resonance**: batch `words[]` arrays to embedding endpoint; compute cosine similarity matrix; identify high-Φ pairs above threshold θ₁
- **Bridge connection generation**: for pairs above θ₁, call `addConnection()` with bridge material; store resonance score and collision type on connection object
- Corpus v1 complete (≥ 150 annotated items)

### M10–M12: Conference + System Test D1
- **Conference 1**: ESWC (Extended Semantic Web Conference), Hersonissos, Crete — primary venue for KG construction and LLM-supported KG research
- **System Test D1**: Multi-vault prototype
  - ≥ 2 discipline graphs loaded simultaneously in NXS+
  - ≥ 20 resonant node pairs detected
  - Bridge connections visible in 3D scene
  - Corpus v1 annotated (≥ 150 items)

## Year 2 — System Build

**Goal**: Build the full agent society, collision engine, and n* spawning protocol with provenance tracing.

### M13–M15: Agent Society
- Agent role formalisation: system prompt design for all 9 roles
- Retrieval filter specification per role
- Memory layer architecture
- Structured output schema for n* records
- Initial testing with 2–3 roles on seed corpus

### M16–M18: Memory, Retrieval, Behaviour Testing
- Memory layer implementation: per-agent scoped context
- Retrieval layer: semantic search across node embeddings
- Stance behaviour testing: does each agent maintain its epistemic function across diverse inputs?
- Anti-drift protocols: ensuring agents don't converge to generic summarisation

### M19–M21: Collision Engine
- **Collision engine** (6 types): Analogy, Contradiction, Grafting, Scale Shift, Ontological Mismatch, Method Transfer
- **n* spawning protocol**: second-order novel nodes generated with provenance tracing via `spawnSphere()`
- **Provenance dashboard**: panel listing all generated n* objects with trace scores
- Ontology construction layer v1: OWL/Turtle export via RDFLib (Python); competency questions generated by LLM

### M22–M24: Colloquium + System Test D2
- **SCC Colloquium**: internal research event with presentation and live demonstration
- **Visiting Professor** residency (5-day): external expert in cultural AI or transdisciplinary research
- **Conference 2**: Ars Electronica Festival or ISEA — positions project within arts-and-technology discourse
- **System Test D2**: Collision engine
  - 9 agent roles active
  - ≥ 3 collision types producing traceable n* objects
  - ≥ 10 evaluated bridge-concepts
  - Ontology layer v1 drafted
  - ≥ 1 faculty identifies a pedagogical use

## Year 3 — Deployment

**Goal**: Full SCC pilot, evaluation rubric deployment, publication, and open dataset release.

### M25–M27: SCC Pilot
- Pilot deployment in SCC seminars and research clusters
- Evaluation rubric applied to growing n* corpus
- Case studies: 3 documented transdisciplinary bridge events
- Interface refinement based on researcher feedback

### M28–M30: Evaluation and Case Studies
- Evaluation rubric fully deployed (6 criteria: Traceability, Disciplinary Distance, Conceptual Coherence, Categorical Novelty, Pedagogical Value, Artistic Usefulness)
- Case study documentation and analysis
- Bridge-concept corpus curation: classification of all evaluated n* objects

### M31–M33: Interface Refinement + Bridge Corpus
- Final interface refinements
- Bridge-concept corpus curation: 30+ evaluated objects
- Method paper draft: graph-based cultural AI
- Open KG dataset preparation

### M34–M36: Conference + System Test D3
- **Conference 3**: Digital Humanities (DH) or ECAI — DH for pedagogical/curatorial outputs; ECAI for the method paper
- **System Test D3**: Deployed observatory
  - Full SCC pilot complete
  - ≥ 30 evaluated bridge-concepts
  - ≥ 1 course or project cites a bridge-concept from the system
  - Method paper submitted / under review
  - Open KG dataset released
